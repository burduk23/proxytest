# SOCKS5 over WebSocket Proxy

Этот проект предоставляет SOCKS5 прокси-сервер, который туннелирует трафик через WebSocket. Это позволяет обходить блокировки, маскируя прокси-трафик под обычный веб-трафик (HTTPS/WSS). Проект состоит из серверной части (для Render) и клиентской части (для вашего VPS на Ubuntu).

## Установка на сервере Ubuntu (для обхода блокировок)

Для того чтобы использовать этот прокси, вам понадобится VPS с установленной ОС Ubuntu. Клиентская часть будет принимать SOCKS5 подключения и перенаправлять их на ваш сервер Render.

### Шаг 1. Установка Node.js 20

Подключитесь к вашему серверу по SSH и выполните следующие команды для установки Node.js 20:

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

Убедитесь, что Node.js установлен корректно:
```bash
node -v
npm -v
```

### Шаг 2. Копирование файлов на сервер

Создайте директорию для проекта на сервере:
```bash
mkdir -p ~/proxy-client
cd ~/proxy-client
```

Скопируйте следующие файлы из этого репозитория на ваш сервер в созданную папку `~/proxy-client`:
- `client.js`
- `package.json`
- `package-lock.json`

**Важно:** Файл `server.js` копировать не нужно, он предназначен только для развертывания на платформе Render.

### Шаг 3. Установка зависимостей

Находясь в директории проекта (`~/proxy-client`), выполните установку необходимых пакетов:

```bash
npm install
```

### Шаг 4. Установка PM2

PM2 — это менеджер процессов для Node.js, который позволит клиенту работать в фоновом режиме и автоматически перезапускаться при сбоях или перезагрузке сервера.

```bash
sudo npm install -g pm2
```

### Шаг 5. Настройка переменных окружения и запуск

Для настройки логина, пароля и других параметров необходимо создать файл `.env` в папке с проектом и добавить в него переменные.

Пример содержимого файла `.env`:
```env
REMOTE_URL=https://ваш-адрес-на-render.onrender.com
SOCKS_USER=ваш_логин
SOCKS_PASS=ваш_пароль
LOCAL_PORT=1080
```

Основные переменные:
- `REMOTE_URL` — адрес вашего сервера на Render (например, `https://proxy-il1y.onrender.com`).
- `SOCKS_USER` — логин для подключения к вашему SOCKS5 прокси.
- `SOCKS_PASS` — пароль для подключения к вашему SOCKS5 прокси.
- `LOCAL_PORT` — порт, на котором будет работать SOCKS5 прокси (по умолчанию 1080).

Запустите клиент с помощью PM2:

```bash
pm2 start client.js --name "proxy-tunnel"
```

Сохраните список процессов PM2, чтобы они запускались после перезагрузки сервера:
```bash
pm2 save
pm2 startup
```
*(Выполните команду, которую предложит `pm2 startup`)*

## Пример настройки прокси в Telegram

После успешного запуска клиента на вашем Ubuntu сервере, вы можете настроить Telegram для работы через этот прокси:

1. Откройте Telegram и перейдите в **Настройки** -> **Продвинутые настройки** -> **Тип соединения** (или **Данные и память** -> **Настройки прокси** на мобильных устройствах).
2. Выберите **Использовать собственную прокси** -> **Добавить прокси**.
3. Выберите тип **SOCKS5**.
4. Заполните поля:
   - **Сервер**: IP-адрес вашего Ubuntu сервера.
   - **Порт**: `1080` (или тот, который вы указали в `LOCAL_PORT`).
   - **Имя пользователя**: значение вашей переменной `SOCKS_USER` (например, `dandon`).
   - **Пароль**: значение вашей переменной `SOCKS_PASS` (например, `pulk`).
5. Сохраните настройки и убедитесь, что статус изменился на "Подключено".
